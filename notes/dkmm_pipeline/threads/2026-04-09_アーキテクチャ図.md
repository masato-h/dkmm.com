> Mermaid と SVG が GitHub Mobile アプリで見えるかのテスト + 実際に役立つ図の資産化。
> 全部レンダリングできれば、今後 thread に図を埋めていける = スマホレビューの解像度が上がる。

## 1. vision pipeline 全体アーキテクチャ (flowchart)

```mermaid
flowchart TD
    A[X で bookmark 追加] -->|Cloud Scheduler<br/>毎日 cron| B[Cloud Run: fetch_bookmarks]
    B --> C[Secret Manager<br/>OAuth refresh_token 読込]
    C --> B2[X API 呼出]
    B2 --> D{新着あり?}
    D -->|なし| E[終了]
    D -->|あり| F[Claude API<br/>または Claude Code Action]
    F -->|draft 生成| G[Slack channel に投稿]
    G --> H[dkm がスマホで通知 受領]
    H -->|気になったら| I[Slack スレッドで反応]
    I --> J[Cloud Run: webhook 受信]
    J --> K[Claude で対話処理]
    K --> L[GitHub repo に commit]
    L --> M[notes/topic/threads/ 更新]
    M -.->|next day| A

    style A fill:#ffe4e1
    style H fill:#ffe4e1
    style I fill:#ffe4e1
    style M fill:#e1f5e1
```

## 2. Slack 対話の sequence diagram

```mermaid
sequenceDiagram
    participant X as X API
    participant CR as Cloud Run
    participant SM as Secret Manager
    participant C as Claude(Code Action)
    participant S as Slack channel
    participant D as dkm (iPhone)
    participant GH as GitHub repo

    Note over CR: Cloud Scheduler が daily で起動
    CR->>SM: refresh_token 読込
    SM-->>CR: token
    CR->>X: bookmarks 取得
    X-->>CR: 99件
    CR->>SM: 新 refresh_token 書戻 (rotate)
    CR->>CR: 既見ID と差分計算
    alt 新着あり
        CR->>C: draft 生成依頼 (関心軸 + 著作権ルール)
        C-->>CR: thread draft + Slack要約
        CR->>S: post 「新着 N 件、ねーさん draft」
        S->>D: 通知 push
        D->>S: スレッドで反応 「これ深掘りして」
        S->>CR: webhook
        CR->>C: 対話処理 (関連 thread を Grep で targeted Read)
        C-->>CR: 結論 + 更新後 thread
        CR->>GH: commit & push
        GH-->>D: PR 通知 (任意)
    else 新着なし
        CR->>CR: 終了
    end
```

## 5. ベッドからの作業フロー (state diagram)

```mermaid
stateDiagram-v2
    [*] --> PC起動
    PC起動 --> RC実行: claude remote-control --spawn worktree
    RC実行 --> ベッド移動
    ベッド移動 --> アプリ接続: iPhone Claude Code app
    アプリ接続 --> 既存セッション選択
    既存セッション選択 --> 作業中
    作業中 --> 新セッション欲しい
    新セッション欲しい --> 新会話作成: アプリで「+」ボタン
    新会話作成 --> auto_spawn: PC側で新 worktree 自動作成
    auto_spawn --> 別作業
    別作業 --> 完了
    別作業 --> 新セッション欲しい: トピック切替
    完了 --> 寝落ち
    寝落ち --> [*]
```
